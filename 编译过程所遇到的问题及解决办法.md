# FourQlib WebAssembly 完整实现指南

## 一、问题链与修复

| # | 问题 | 表现 | 修复 |
|---|------|------|------|
| 1 | `build.sh` 使用 Windows CRLF 换行符 | bash 无法解析反斜杠续行符 | 重写为 LF 格式 |
| 2 | 缺少 `fourq_wasm.c` 入口文件 | 编译缺少导出函数 | 创建入口文件，导出 `wasm_FullKeyGeneration` 等 |
| 3 | `random_bytes` 重复定义 | `duplicate symbol: random_bytes` | 从 `build.sh` 移除 `../random/random.c` |
| 4 | `schnorrq.c` 引用 `<malloc.h>` | WASM 中不存在，触发 `unreachable` | 改为 `<stdlib.h>` |
| 5 | `random_bytes` 返回 `void` | 状态码检查失败 | 改为返回 `int`，成功返回 `1` |
| 6 | 未导出 `_malloc`/`_free` | `Module._malloc is not a function` | 添加 `EXPORTED_FUNCTIONS` |
| 7 | `file://` 协议阻止 WASM | 浏览器安全限制 | 用 `python3 -m http.server` 启动服务 |

## 二、运行时哈希切换实现

### 原理
用全局函数指针替换编译期宏定义，实现零开销动态切换。

### 修改 `schnorrq.c`
```c
#include <stdlib.h>
#include <string.h>

// 在 #include 之后添加：
extern int (*current_hash_fn)(const unsigned char *, unsigned long long, unsigned char *);
#undef CryptoHashFunction
#define CryptoHashFunction current_hash_fn
```

### 修改 `fourq_wasm.c`
```c
#include <emscripten.h>
#include <stdio.h>
#include <string.h>
#include <stdlib.h>
#include "FourQ_api.h"
#include "../sha512/sha512.h"
#include "blake2b/blake.h"

// 默认使用 SHA-512
int (*current_hash_fn)(const unsigned char *, unsigned long long, unsigned char *) = crypto_sha512;

EMSCRIPTEN_KEEPALIVE
void set_hash_algo(int algo) {
    current_hash_fn = (algo == 1) ? crypto_blake2b : crypto_sha512;
}

int random_bytes(unsigned char* buf, unsigned int len) {
  unsigned int i;
  for (i = 0; i < len; i++) {
    buf[i] = (unsigned char)(rand() & 0xFF);
  }
  return 1;
}

EMSCRIPTEN_KEEPALIVE
int wasm_FullKeyGeneration(unsigned char* sk, unsigned char* pk) {
  return SchnorrQ_FullKeyGeneration(sk, pk);
}

EMSCRIPTEN_KEEPALIVE
int wasm_Sign(const unsigned char* sk, const unsigned char* pk,
              const unsigned char* msg, unsigned int msg_len,
              unsigned char* signature) {
  return SchnorrQ_Sign(sk, pk, msg, msg_len, signature);
}

EMSCRIPTEN_KEEPALIVE
int wasm_Verify(const unsigned char* pk, const unsigned char* msg,
                unsigned int msg_len, const unsigned char* signature,
                unsigned int* valid) {
  return SchnorrQ_Verify(pk, msg, msg_len, signature, valid);
}
```

### 修改 `build.sh`
```bash
#!/bin/bash
emcc -O3 -s WASM=1 -s ASSERTIONS=1 \
     -s EXPORTED_RUNTIME_METHODS='["cwrap","ccall","HEAPU8","HEAP8","getValue","setValue"]' \
     -s EXPORTED_FUNCTIONS='["_malloc","_free","_wasm_FullKeyGeneration","_wasm_Sign","_wasm_Verify","_set_hash_algo"]' \
     -s MODULARIZE=1 -s EXPORT_NAME='createFourQModule' \
     -s ALLOW_MEMORY_GROWTH=1 -s SINGLE_FILE=0 \
     -I. -I../sha512 -I../random -I./blake2b \
     fourq_wasm.c schnorrq.c kex.c eccp2.c eccp2_core.c crypto_util.c hash_to_curve.c \
     ../sha512/sha512.c blake2b/blake.c \
     -o fourq.js \
     -D _GENERIC_ -D _AMD64_ -D __LINUX__ -D USE_ENDO
echo "✅ 编译完成！生成 fourq.js 和 fourq.wasm"
```

## 四、编译与运行

```bash
# 1. 激活 emsdk
source ~/emsdk/emsdk_env.sh

# 2. 编译
cd ~/FourQlib/FourQ_64bit_and_portable
bash build.sh

# 3. 启动 HTTP 服务器
python3 -m http.server 8080

# 4. 浏览器访问 http://localhost:8080/index.html
```

## 五、关键注意事项

1. **必须使用 HTTP 服务器**，不能直接用 `file://` 打开 HTML
2. **`random_bytes` 必须返回 `int`**（成功返回 `1`）
3. **`schnorrq.c` 必须用 `<stdlib.h>`**，`<malloc.h>` 在 WASM 中不存在
4. **必须导出 `_malloc`/`_free`**，否则 JavaScript 无法分配 WASM 内存
5. **`set_hash_algo` 必须加入 `EXPORTED_FUNCTIONS`**，否则前端无法调用
6. 编译时必须同时包含 `../sha512/sha512.c` 和 `blake2b/blake.c`
7. 切换哈希算法后，**已生成的密钥对签名与验证必须使用相同算法**
